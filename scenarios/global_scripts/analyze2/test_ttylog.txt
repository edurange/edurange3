[1mRATIONALE[0m[m
       The  use  of  the [4mCDPATH[24m was introduced in the System V shell. Its use is analogous to[m
       the use of the [4mPATH[24m variable in the shell. The BSD C  shell  used  a  shell  parameter[m
       [4mcdpath[24m for this purpose.[m
[m
       A  common extension when [4mHOME[24m is undefined is to get the login directory from the user[m
       database for the invoking user. This does not occur on System V implementations.[m
[m
       Some historical shells, such as the KornShell, took special actions when the directory[m
       name  contained  a  dot-dot  component, selecting the logical parent of the directory,[m;1705265115
       rather than the actual parent directory; that is, it moved up one level toward the [1m'/'[0m[m
       in  the  pathname, remembering what the user typed, rather than performing the equiva‚Äê[m
       lent of:[m
[m
           [1mchdir("..");[0m[m
[m
       In such a shell, the following commands would not necessarily produce equivalent  out‚Äê[m
       put for all directories:[m
[m
           [1mcd[0m [1m..[0m [1m&&[0m [1mls[0m      [1mls[0m [1m..[0m[m
[m
       This  behavior is now the default. It is not consistent with the definition of dot-dot[m
       in most historical practice; that is, while this behavior has been  optionally  avail‚Äê[m
       able  in  the KornShell, other shells have historically not supported this functional‚Äê[m
       ity. The logical pathname is stored in the [4mPWD[24m environment variable when the [4mcd[24m  util‚Äê[m
       ity  completes  and  this  value is used to construct the next directory name if [4mcd[24m is[m
       invoked with the [1m‚àíL[0m option.[m

[1mDESCRIPTION[0m[m
       [1mNetstat[0m prints information about the Linux networking subsystem.  The type of informa‚Äê[m
       tion printed is controlled by the first argument, as follows:[m
[m
   [1m(none)[0m[m
       By default, [1mnetstat[0m displays a list of open sockets.  If you don't specify any address[m
       families, then the active sockets of all configured address families will be printed.[m
[m
   [1m--route[0m [1m,[0m [1m-r[0m[m
[7m Manual page netstat(8) line 1 (press h for help or q to quit)[27m[K
[K [KESCESC[K[[[K66[K~~
[K       Display the kernel routing tables. See the description in [1mroute[0m(8) for details.   [1mnet‚Äê[0m[m
       [1mstat[0m [1m-r[0m and [1mroute[0m [1m-e[0m produce the same output.[m
[m
   [1m--groups[0m [1m,[0m [1m-g[0m[m
       Display multicast group membership information for IPv4 and IPv6.[m
[m
   [1m--interfaces,[0m [1m-i[0m[m
       Display a table of all network interfaces.[m
[m
   [1m--masquerade[0m [1m,[0m [1m-M[0m[m
       Display a list of masqueraded connections.[m
[m
   [1m--statistics[0m [1m,[0m [1m-s[0m[m
       Display summary statistics for each protocol.[m

[1mOPTIONS[0m[m
   [1m--verbose[0m [1m,[0m [1m-v[0m[m
       Tell the user what is going on by being verbose. Especially print some useful informa‚Äê[m
       tion about unconfigured address families.[m
[m
   [1m--wide[0m [1m,[0m [1m-W[0m[m
       Do not truncate IP addresses by using output as wide as needed. This is  optional  for[m
       now to not break existing scripts.[m
[m
   [1m--numeric[0m [1m,[0m [1m-n[0m[m
       Show  numerical  addresses  instead of trying to determine symbolic host, port or user[m
       names.[m
[m
   [1m--numeric-hosts[0m[m
       shows numerical host addresses but does not affect the  resolution  of  port  or  user[m
       names.[m
[m
   [1m--numeric-ports[0m[m
       shows numerical port numbers but does not affect the resolution of host or user names.[m
[m
   [1m--numeric-users[0m[m
       shows numerical user IDs but does not affect the resolution of host or port names.[m
[m
   [1m--protocol=[0m[4mfamily[24m , [1m-A[0m[m
       Specifies  the  address families (perhaps better described as low level protocols) for[m
       which connections are to be shown.  [4mfamily[24m is a comma (',') separated list of  address[m
       family keywords like [1minet[0m, [1munix[0m, [1mipx[0m, [1max25[0m, [1mnetrom[0m, and [1mddp[0m.  This has the same effect[m
       as using the [1m--inet[0m, [1m--unix[0m ([1m-x[0m), [1m--ipx[0m, [1m--ax25[0m, [1m--netrom[0m, and [1m--ddp[0m options.[m
[m
       The address family [1minet[0m includes raw, udp and tcp protocol sockets.[m

[K   [1m-c,[0m [1m--continuous[0m[m

[m
   [1m-e,[0m [1m--extend[0m[m
       Display additional information.  Use this option twice for maximum detail.[m
[m
   [1m-o,[0m [1m--timers[0m[m
       Include information related to networking timers.[m
[m
   [1m-p,[0m [1m--program[0m[m
